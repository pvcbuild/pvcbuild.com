Get started with PVC using Chocolatey.

cinst pvc

Install some plugins.

pvc install Pvc.Less; pvc install Pvc.MSBuild

Then create a pvcfile.csx

pvc.Task("default", () => {
    pvc.Source("*.less")
       .Pipe(new PvcLess())
       .Save("~/deploy");
});

pvc.Task("build", () => {
	pvc.Source("WebApplication.sln")
		.Pipe(new PvcMSBuild(
			buildTarget: "Clean;Build"
	        enableParallelism: true
		));
});

And let the magic happen (compile your LESS):

pvc

Or to trigger MSBuild:

pvc build

Check out the documentation to learn more about how to build your pvcfile.

PVC's pvcfile.csx is a C# script powered by scriptcs. Make a change, run your script. No compilation, No IDE necessary. Just get things done!


Define Tasks, call plugins or handle streams yourself, right in your script.

pvc.Task("list-projects", () => {
    pvc.Source("**/*.csproj")
       .Pipe((streams) => {
       		foreach (var stream in streams) {
       			Console.WriteLine(stream.StreamName);
       		}
       });
});

Plugins

Plugins provide the functionality that makes PVC nice to use. Like we saw in the Getting Started section, you can install plugins from the command line using pvc install Pvc.PluginName

Here's a selection of PVC plugins
Pvc.MSBuild Build your project with PVC.
Pvc.NuGet Build and push packages to NuGet and SymbolSource.
Pvc.Less Compiles LESS to CSS and creates source maps.
Pvc.Sass Compiles SASS to CSS and creates source maps.
Pvc.Ajaxmin Minifies JavaScript and generates sourcemaps.
Pvc.HtmlCompressor Minifies HTML.
Pvc.S3 Upload files to Amazon S3.
Pvc.AzureBlob Upload files to Microsoft Azure Blob Storage.
Pvc.Browserify Runs Browserify (yep, from Node!) on your JS.

Scaffolding

PVC supports scaffolding new applications and components using pvc init template-name


// Sample from https://github.com/pvcbuild/pvc-init-templates/blob/master/plugin.json
{
    "templateRepo": "https://github.com/pvcbuild/pvc-init-plugin.git",
    "replacementExtensions": [".cs", ".csproj", ".nuspec", ".sln", "LICENSE"],
    "config": {
        "pluginName": null,
        "license": null
    },
    "prompts": {
        "Plugin Name: ": {
            "field": "pluginName"
        },
        "License: ": {
            "field": "license",
            "options": [
                ["MIT"],
                ["Apache 2", "Apache2"],
                ["BSD"],
                ["None (will add later)", null]
            ],
            "choose": "LICENSE"
        }
    }
}

A template consists of a simple JSON file that links to a git repository, configuration variables and prompts. To add your own templates make a pull request against the pvc-init-templates repository.

Build a Plugin

Extending and creating a plugin in PVC is incredibly simple. Start out by running the init script: pvc install plugin


Your plugin class will extend PvcPlugins.PvcPlugin and implement the streaming interface. 

public override IEnumerable Execute(IEnumerable inputStreams)
{
    var resultStreams = new List<PvcStream>();

    foreach (var inputStream in inputStreams)
    {
        // Do things with Streams, add to results
        resultStreams.Add(inputStream);
    }

    return resultStreams;
}

A plugin can filter the streams passed to it by overriding the SupportedTags property. 

// Only pass in LESS streams
public override string[] SupportedTags
{
    get
    {
        return new string[] { ".less" };
    }
}

Every plugin in the pipeline has full control to add or remove any streams at will. If you're interested in building a plugin, checkout our plugin request list for ideas!
